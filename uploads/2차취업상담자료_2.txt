1차프로젝트 주제 : 푸디버디 ( 맛집리뷰와 모임 커뮤니티)

- 역할 : 관리자

- 담당 기능 :           1. 회원 목록 조회 및 상세내역 수정
                           2. 모임게시판 모임결성글 승인 및 반려 
                           3. qna게시판 답글 
                           4. 공지사항 게시판 관리 ( 등록,수정,삭제)

- 주요 사용 기술 :     DBMS  :   MariaDB - HeidiSQL 
           		   Server :    Apache Tomcat 9	
                                        Servlet, JSP,  jQuery, JSTL, SweetAlert, Bootstrap


- 배운 점 ? 깨달은점?

 MVC 구조 이해: Model, View, Controller의 역할 분담과 데이터의 흐름을 파악할 수 있게되었고 ,  패키지구조나 클래스를 구성하고 설정하는 방식을 배웠습니다.

 Servlet을 통해 클라이언트 요청을 동기적으로 처리하고, 그 결과를 다시 JSP 페이지로 리다이렉트  하는 흐름을 익혔습니다. 


- 어려웠던 점 (기술적) :

 대량의 데이터를 효율적으로 처리하고 화면에 페이징하는 과정에서 SQL의 LIMIT와 OFFSET을 적용하는 방법을 학습하고, 페이지 번호를 계산하는 로직 구현에 어려움이 있었습니다. 총 데이터 수와 페이지 당 데이터를 나눌 때 정확히 페이지 수를 계산하지 않으면, 페이지 번호가 더 많이 나오거나 적게 나올 수 있으며, 마지막 페이지에 남은 데이터를 정확히 보여주지 못할 수 있습니다.

상태 관리측면에서  모임 게시판의 결성글 승인 및 반려 기능에서 글의 상태를 관리하고, 변경된 상태에 따라 다른 페이지로 이동시키는 로직 구현이 어려웠었습니다.
 
- 아쉬웠던 점 (기술적/감정적) : 동기식 처리 방식에서 화면 전체를 리프레시하는 방식으로 데이터 처리 로직이 구성되어 있어, 사용자 측면에서 비효율적이고 보기에도 안좋은 부분이 있었습니다.. 특히 회원 목록 조회나 게시글 관리에서 화면 전환 없이 데이터를 갱신하는 비동기 방식의 필요성을 느꼈습니다.

부트스트랩을 적용하고 css를 수정하는 부분에서 경로설정이나 기능구현에 시간을 많이 쏟은 탓에 화면설계대로 완벽히 보기좋은 화면을 구성하지 못한게 아쉬웠습니다.





2차 프로젝트 주제 :  중소기업 그룹웨어

- 역할 :  로그인과 마이페이지 구성 , 근태관리와 문서함 기능 구현 


- 담당기능 :           1. 사원 로그인 기능 
                          2. 개인정보 조회
                          3. 출퇴근 , 연차 ,근무내역 관리 
                          4. 문서함 

- 기대하는 바  / 목표 : 

현재 진행 중인 중소기업 그룹웨어 프로젝트는 AJAX와 JSON, Spring Boot(Gradle)를 활용하여 개발하고 있으며, 이와 함께 Spring Security를 통한 웹 보안과 권한 설정을 도입하고 있습니다. 
프로젝트의 핵심 목표는 근태 관리와 문서함 기능을 포함한 그룹웨어 시스템을 구현하면서, 안전하고 효율적인 사용자 인증 및 권한 관리 기능을 제공하는 것입니다.

AJAX와 JSON을 활용하여 페이지 리로드 없이 비동기적으로 데이터를 처리함으로써 사용자의 환경을 개선하고, 실시간 출퇴근 관리와 연차 및 근무 내역 조회 기능을 통해 반응성이 빠르고 직관적인 시스템을 구축할 예정이고 ,Gradle을 이용해 의존성 관리를 효율적으로 수행하며, Spring의 핵심 기능인 의존성 주입을 통해 백엔드 개발 역량을 강화할 것입니다.

특히, 웹 애플리케이션에서 Spring Security를 사용하여 로그인, 로그아웃, 사용자 인증 및 권한 부여 과정을 견고하게 구현할 것이고, 이를 통해 각 사원의 권한에 따라 시스템 접근 범위를 다르게 설정하고, 특정 기능(예: 관리자 기능이나 중요한 문서 관리 기능)에 대해 접근 제어를 효과적으로 구현할 것입니다.
